# probe

目的：

以苏格拉底式口试，检验一个人对某个 library 的掌握是「死记硬背」还是「真正理解」，并给出可执行的改进反馈。

本命令的设计哲学是：「问出对的问题，就已经解决了问题的大半；最怕的是找不到该怎么问。」因此 probe 的核心价值不在打分，而在于先从库内提炼出真正值得问的问题——能鉴别背诵与理解、能融汇全库精华、又能把作答者逼出库外的那几道问题。问对了题，考核本身就成立了一大半。

需要鉴别的对手不是「背了 index 摘要的人」，而是「背熟了 `wiki/sources/` 逐源摘要的人」。库内的逐源摘要往往已经写满了机制、边界、原文引用，甚至 lint 已把两源的相关模块以二级标题锚点互链、并附「为什么相关」的说明——这意味着机制层、边界层、跨源张力的现成措辞，库里已经替背诵者准备好了。因此 probe 的题目和判据都必须默认对手能流利复述任何单个 source 段落（含其自带的边界与反例、含 lint 互链说明），真正的鉴别力只能来自「逼作答者离开库内任何单一段落已铺好的轨道、自行推导/迁移/反推」的地带。

probe 给出的题目必须同时满足三大硬要求：

- 硬要求①·能鉴别「死记硬背 vs 真正理解」：题目不能靠复述原文措辞、原书例子、index 摘要或任一 source 段落（含其互链说明）就答对；凡库内已成段写明的机制、边界、张力，一律不得直接作为得分点，必须再下沉一层到 source 未明写、需作答者自行推导/迁移/反推的层级。
- 硬要求②·必须融汇库内知识精华：题目要做跨源、跨章的综合（识别张力、对撞两源、整合谱系），而不是单点的表层事实复述。
- 硬要求③·不能仅局限于库内：题目必须有一脚踏出库外——迁移到库内没出现过的同构新情境、要求自造库外正例/反例、与库外知识对照、或批判该库自身的盲区与时代/立场局限。

命令格式：

- `probe [libraryName] [可选:关注的方面]`

示例：

- `probe investWiki`
- `probe aiagentWiki 记忆与检索`

输入：

- 一个 `libraryName`
- 一个可选的「关注的方面」（主题/章节/某个观点）

输入要求：

- 必须显式提供 `libraryName`，不能省略。如果省略，必须提示用户补充，否则不能进入后续步骤。
- 「关注的方面」可选。如果提供，则把出题范围收敛到该方面；如果不提供，则覆盖全库精华骨架。
- probe 默认在指定 library 范围内读取与出题，作答者是与本命令交互的用户。

## 执行总览：三阶段架构

probe 分三个阶段执行。**性能关键**：全部重活（读全库、建地图、出题、写判据）都在阶段〇一次性完成并落盘缓存；阶段一的每一轮口试只加载「口试轮热协议」+ 当题判据卡，绝不重读库、不重建判据。

| 阶段 | 动作 | 需要读取的规则 |
|---|---|---|
| 〇·备考（用户无感知） | 缓存检查→建库内地图→构建题库→写缓存 | 本文件全部「备考期」章节 |
| 一·口试（多轮交互） | 一次一题、依答追问、判级收口 | 仅「口试轮热协议」+ 当题判据卡 |
| 二·报告 | 综合评判报告 | 「报告期」章节 + 题库缓存全文 |

## 阶段〇·备考（对用户无感知，产出题库缓存）

### 0.1缓存检查（先做，命中则跳过整个备考）

1. 缓存文件路径：`[libraryName]/wiki/.probe/bank-[方面slug|全库].md`（「方面slug」直接用用户给的方面原词；未给方面则为 `全库`）。
2. 若缓存存在：读取其头部「源清单与哈希」，对当前 `wiki/sources/*.md`（含范围内 `wiki/fragment/*.md`）逐一重新计算哈希（`shasum -a 256` 取前 12 位）比对。全部一致且无新增/删除文件 → **直接复用缓存，跳到阶段一开考**；任何不一致 → 静默重建（走 0.2 起的完整备考）。
3. 缓存不存在 → 走完整备考。
4. 对用户只需一句进度提示（如「正在备考出题…」），不展示备考过程与题库内容。

### 0.2存在性检查与范围收敛

- 若 `[libraryName]/wiki/` 不存在，立即终止并提示用户检查库名。
- 若用户给了「关注的方面」，把库内地图与题库收敛到该方面；若按 query 规则检索后该方面在库内无任何相关素材，必须如实告知该方面库内无沉淀，并询问用户是改为全库出题、还是补充资料后再 probe，不擅自硬出。

### 0.3建「库内地图」

按 query 的源读取规则：优先读 `wiki/sources/` 与 `wiki/fragment/` 的正文并沿其中的交叉链接续读相关页面；`wiki/index.md`（含主题索引）仅用作定位入口与覆盖面核对，不得停留在其摘要层取材；**不要参考 `wiki/query/` 下的内容**；当 wiki 不足以支撑出题时，可读 wiki 对应的 raw 原文核对细节。读完后列出本库的「精华骨架」：核心原则、跨源张力点（彼此冲突或互证的论断）、各源自标的边界/陷阱/反例、库已自曝的盲区与时代/立场局限。

- 精华甄别（去噪）：先给源分层，按「一手原典/系统方法论 > 二手解读 > 个人经验/转写稿（尤其标 `Confidence: Medium`、whisper 自动转写、自述战绩类）」排序。跨源张力优先在高可信层之间构造；当不得不用低信号源出题时，把「该源可信度/立场偏差/未经审计」本身设为考点（接批判型题型）。「融汇了哪些精华」栏须注明每条所引源的可信层级，不得把声量最大的口水观点当精华对撞。
- 库容自检与降级（先判库容，再定硬要求②口径）：
	- ≥2 个实质性源且有真实张力 → 按跨源口径正常出题。
	- 单源/源极薄 → 硬要求②降级为「跨章/跨小节的张力与综合」，并在报告中声明「本库为薄库，②按跨章口径执行」。
	- 源数与可信结论不足以支撑 3 道合格题 → 判定「本库暂不适合 probe」，回退 `需要创建新 Wiki 页面（New Wiki Page Needed）` + `Confidence: Low`，不得硬造张力。

### 0.4构建题库（3-7道主干题，每题一张完整判据卡）

基于库内地图设计 3-7 道主干题，覆盖不同维度、不同题型、不同源组合，不要同质。每道题的判据卡必须预先写清以下七项，并自检满足三大硬要求：

1. 题面：钉死在一个具体机制或一处跨源（薄库时跨章）张力上，不写成无锚点的空泛大题。
2. 融汇了哪些精华（对应硬要求②）：列出综合了哪几源、哪几章、哪条张力，并标注可信层级。
3. 出库钩子（对应硬要求③）：迁移到哪个库外新情境、要求造哪类库外新例/反例、引入哪类库外知识对照、或批判该库哪个盲区。钩子必须「同构且自足」：迁移情境须与所考原则结构同构，真懂即可完成，不依赖库外专有冷知识；批判/对照类所需库外常识必须是受过基础训练者普遍可得的，否则改为「要求作答者自己举库外例/反例」。自检：「这题答错，是因为不懂库内原则，还是仅因为不知道某条库外事实？」——若是后者，判废重出。
4. 死记答案长什么样：预判背诵者贴着原文/原例/某 source 段落的回避式答案。
5. 真懂答案关键点：真懂者必须命中的机制层/边界/综合/迁移要点。**不得是某个 source 段落的压缩复述——必须含至少一项「该段落未覆盖、需作答者推导或迁移」的内容**，否则该题不合格、重出。
6. 追问钻头（2-4个下钻方向）：①追机制（你说X，为什么？再为什么？）②试边界（什么情况下X不成立？）③要新例（举个库里没有的例子）④推反事实（前提反过来呢？）。不预设逐字脚本——实际追问必须由作答者上一句的具体内容生成。
7. 证伪台阶设计：预设一个看似合理实则错误的诱导说法、或与预期立场相反的强论据。护栏：只在「现象/情境/类比」层面构造，不得点出正确机制名或结论方向；自检「即便作答者顺着诱导走，也不会白得正解关键词」。

- 库内锚定（防「聪明门外汉」）：至少一题必须考库内特有、非公共常识的内容（如某源自标待核实/被推翻的声明、两源在某具体问题上的真实分歧），在该题判据卡上标注 `库内锚定:是`。
- 题型清单（相邻两题尽量不同型，无需覆盖全部）：迁移到全新情境（Transfer）／跨源对撞·找张力（Cross-source Collision，薄库改同源内部张力）／边界与失效条件（Boundary）／反事实·反过来想（Counterfactual）／追到机制层问为什么（Mechanism / 5-Whys）／自造新例·反例（Generate Example）／批判库内某论断（Critique the Library）／两难裁决（Dilemma）。
- 禁出题型：可直接复述 index 摘要、二级标题序列、原书例子或任一 source 段落（含 lint 互链说明）就能答对的题；是非题/二元封闭题；纯记忆琐事/可检索事实题；纯术语定义连连看；无锚点空泛大题；一口气塞 4 问的复合多跳题；答案藏在题干里的自带提示题；诱导性预设立场题；把评分维度写进题面的提问即泄底题。

### 0.5写入题库缓存

格式（写入 `[libraryName]/wiki/.probe/bank-….md`）：

```
# probe题库缓存（作答者请勿阅读——阅读将使口试结果失效）
- library: <libraryName>
- 方面: <方面|全库>
- 源清单与哈希:
	- sources/<文件名>.md <sha256前12位>
	- （范围内的 fragment 同列）
- 库容判定: 正常 | 薄库(②按跨章口径) 

## 库内地图
<精华骨架/张力点/源可信分层/库已自曝盲区>

## 题1·<题型>·<短标题>
<七项判据卡 + 库内锚定标记>

## 题2·…
```

- `.probe/` 目录仅由 probe 命令读写；缓存内容在口试结束前**绝不向作答者展示任何片段**。
- 泄题说明：缓存是用户自己库里的文件，防君子不防己；文件头警示已尽提示义务。

## 阶段一·口试（多轮交互）

### 口试轮热协议（每轮唯一需要遵循的规则集）

1. **一次只抛一题**。从难度适中的题（迁移题或机制题）开抛；抛出后立即结束本轮回复、交还控制权，等待作答。不预告后续题目、不剧透、不自问自答。收到作答前不得继续下一题。
2. **收到作答后，只加载当题判据卡与本协议**——不重读库、不重建判据、不翻其他题的卡。
3. **听答定位破绽**：拿回答逐条比对卡上「死记答案长什么样 / 真懂答案关键点」，记下最值得下钻的一个具体破绽（优先：口号无机制、迁移失败、看不出边界、造不出新例、答案与某 source 段落高度同构）。
4. **依答追问下钻**：从卡上追问钻头中选一个方向，追问措辞由作答者上一句的具体内容生成，一次只钻一个点。追问下钻属同题内动作、不占题目配额。
5. **每题恰做一次证伪台阶**：抛出卡上预设的诱导说法，看对方被带跑还是用判据反驳；不反复施压把答案逐步漏出。
6. **判官自律（四条）**：①出处归因——只计作答者明说的，替他推断的一律按未答；②不替补全——「说了能让我联想到机制X的话」≠「说了机制X」，须用一次中立追问坐实；③锚点是因果链不是术语——用词与预想判据不符但因果链自洽、可独立复核、能迁移，应判真懂；④防误杀——无法分辨「不懂」还是「懂但说不清」时，给一次「换个说法/举个例子」的中立机会（不喂答案），仍说不清才降级。
7. **防泄露**：追问与回应中不嵌入会泄底的关键词、机制名或结论；不评论对错、不透露「接近了/答反了」；作答者反问套答案，只回「由你作答，评判在结束后给出」；整场不向作答者读出库内 source 原文段落或 index 摘要（引用原文只出现在报告期）。需要定义/前提时让作答者自己给出并据此作答。
8. **判级收口**：对照卡上要点即时内部记录本题暂定级 L0-L4 与判据（不告知作答者），一句话内部记下关键缺口。L2/L3 分界线：判 ≥L3 前，至少有一次追问把作答者推到「库内任何单一 source 段落都未覆盖的地带」（全新情境迁移、或反推库内未写明的前提），并且其在该地带仍能自洽推导；一离开库内已写好的轨道就失抓手，无论前面撑了几层都判 L2。
9. **进入下一题**：换不同题型，并尽量承接上题暴露的薄弱处做针对性进阶（上题机制薄弱→下题选需要机制支撑的两难或反事实）。

### 口试轮速查（遇到再查，不必每轮通读）

- 抗作弊反制：背逐源摘要→强制应用到 sources 链接里没有的第三情境或反推前提；背金句→禁原句复用、追因果链；堆术语→对一个术语做「定义-边界-取舍」三连+区分最易混近义词；复述摘要骨架→打乱顺序抽脚手架问内部；话题劫持→点破「我问的是Y不是X」锚定原题；万能空话（"要权衡""看场景"）→给两个具体场景各逼一个决定性判断；伪精确数字→追「为什么是这个量级」；诉诸权威→请其批判该做法的盲区并举不适用反例；选择性遗忘冲突→摆出两处冲突要求抽统一原则；反问控场→「定义由你给，异议答完再说」，反问计未作答；元话术冒充元认知→「现在就当场用第一性原理推一遍」。
- 考官陷阱自查：不被流利度/术语密度催眠（卡顿但真推理 > 流畅但背诵）；不因开头答得好就对后续手软；听到与库内一字不差的原话越要警惕、要求改写与迁移；每题独立判定、理解不跨题转移；珍视诚实的「不知道」；每题至少追到一个可证伪的具体判断或一个反例才算考完。
- 死记破绽信号（命中多条偏死记）：贴 source 段落复述、超出该段落即失抓手；只给结论无机制、追「为什么」第一层就同义反复；原则当万能公式、说不出边界反例；造不出库外新例只改名字；跨源冲突两边都背还说「都对」；前提一改推不动；无法批判库内任何内容；两难和稀泥；堆术语说不出代价；只会列举不会综合；记住数字符号却用错条件。
- 真懂标志信号（命中多条偏真懂）：能剥离原例迁移并说清成立条件；每层「为什么」都下沉到更底层可独立检验的理由；主动说出边界、前提与失效预警；能原创经得起定义检验的库外新例/反例；跨源冲突能指出各自成立前提并说该听谁；反事实下能重推结论并点出开关变量；能跳出库批判其盲区并用库外证据佐证；两难明确选边、给判据权重、承认放弃了什么；用自己的话重述更简洁抓要害带代价；主动暴露不确定性；能把多源综合成连贯判断框架；**表达朴素但因果链可独立复核、追问下逐步逼近正解（是真懂迹象，不是卡壳）**。
- 库内锚定信号：能就库内特有、非公共常识的事实/张力作答才算锚定本库；通用原理侃侃而谈却答不出库特异内容→报告中标「疑似未读库/纯先验作答」；高思辨+低库内锚定判「理解力强但未掌握本库」，纯库外通用推理即便漂亮也不足以单独支撑 L3 以上。

## 阶段二·报告（全部题目结束后）

此阶段可重读题库缓存全文与必要的 source 原文。作为完整考官给出「综合评判报告」，简体中文，结构见「输出结果」。

- 综合等级取各题稳健中位（不被单题高光或失手主导），单列「最弱维度」作为改进抓手；必须报告等级分布与离散度：分布双峰或跨度 ≥2 级时，不能只给中位数，要判「局部掌握/掌握不均」并点出哪些维度真懂、哪些死记。
- L0-L4 分级标准：
	- L0·空答/跑题：未触及考点，或只重复题面与无关口号。
	- L1·死记硬背：能复述结论/定义/例子/某 source 段落，但追问机制 2 层内同义反复或卡壳、迁移套不上、说不出边界与反例、造不出库外新例，一旦离开库内已写好的轨道即失抓手。
	- L2·部分理解：机制能下沉一层但到不了底，或边界/迁移/造例三项只做到一项；反向施压会动摇但能部分守住；一离开 source 已铺好的轨道就失抓手。
	- L3·真正理解：稳定给出机制（多层为什么）+ 边界（失效条件）+ 跨源（薄库则跨章）整合，能在库内任一单一段落未覆盖的地带自洽推导并自造合格新例，反向施压下用判据稳住，且有库内锚定。
	- L4·批判性掌握（超越库）：在 L3 之上还能批判该库/该论断的盲区与时代/立场局限、指出库外更优或互补视角、并在两难中做出有判据的裁决。
- 若库内素材不足以支撑某维度出题，必须如实说明，按需提出 `需要创建新 Wiki 页面（New Wiki Page Needed）`。
- 报告后按需提示：本次题库已缓存，库内容不变时下次 probe 将直接开考；若想换一套题可让 probe 重建（删除对应 bank 文件即可）。

## 硬性约束

- 防泄露：作答前绝不展示标准答案、要点清单、评分量规、维度划分或题库缓存的任何片段；判定与改进反馈只在全部作答封口后一次性给出，此前任何泄露都使该题作废重出（重出须换新题面，不得复用已泄题）。
- 一次只抛一题，禁止在单轮内模拟或代答整场口试。
- 题目素材必须真实出自库内（题面可超出库，但所对撞、所批判的库内论断必须确有其源）；不得编造库中不存在的内容、不得伪造来源或链接。
- 每题必须有一脚踏出库外，出库钩子必须「同构且自足」；硬要求②默认跨源口径，薄库降级跨章并在报告中声明。
- probe 测的是「对本库的掌握」：至少一题考库内特有内容做库内锚定。
- 分析所用方法论必须基于库内（库内地图）；题目本身可以、且应当超出库内。
- 任何作答都不应该要求回答者熟悉或完全了解库内的内容：问题基于库内知识提问，但不能假定受试者是照着库内知识作答的。
- 涉及 wiki 页面名称、页面摘要或新增页面建议时，必须使用简体中文；整份评判报告使用简体中文。
- 若最终判定存在任何不确定（题量不足/作答信息不足/素材自相矛盾/判据不充分等），必须标注 `Confidence: Low` 并说明来源。
- 禁止执行复制、剪切、移动或重命名 raw 文件的操作；probe 除写 `wiki/.probe/` 缓存外不修改 wiki。
- 本 spec 尾部「校准示范」仅供备考期校准难度，禁止向作答者展示任何片段、禁止直接复用原题。

## 输出结果

- 一份逐题深挖的苏格拉底口试过程（一次一题、依答追问、多轮交互，过程中不泄露答案）。
- 一份「综合评判报告」（简体中文），至少包含：
	- 逐题等级：每题 L0-L4，并标注判据来自哪几条死记/真懂/库内锚定信号。
	- 总体判定：综合等级（稳健中位）+ 等级分布与离散度 + 一句话「死记硬背 vs 真正理解」总判断；分布双峰或跨度 ≥2 级时改判「局部掌握/掌握不均」。
	- 最强环节与最弱环节；命中「疑似未读库/纯先验作答」或「理解力强但未掌握本库」时单独标注。
	- 针对性改进/精读建议：哪些点要从背口号转为补机制、补边界、练迁移；指出库内最该重读的 `wiki/sources/`、`wiki/fragment/` 页面。
	- 必要时附 `需要创建新 Wiki 页面（New Wiki Page Needed）` 与 `Confidence: Low`。

## 校准示范（仅供备考期校准难度，不得展示或复用）

取自 investWiki 的一道完整样题。

- 题面：「库里同时收着两条对立的赚钱观：一边是段永平/Graham 那套『安全边际是约束条件、85% 的人该买指数』，另一边是 Serenity 自述两年 22561%、靠『在官方确认前买入、信息合成赚认知差』猎卡脖子小盘股。请你不要复述两套方法，而是回答一个问题：这两套在『alpha 从哪来』这件事上，到底是互相矛盾，还是其实说的是同一件事？如果让段永平来评价 Serenity 的打法，他会说这是『投资』还是『投机』，依据库内他自己的哪条标准？」
- 满足硬要求①：用「表面对立的两源是否同根」逼作答者抽象出底层机制，而非平行罗列。背诵者只能各背一段 source 摘要，真懂者能看穿两者共享「可公开=无优势、alpha 必来自不可复制的认知差」这一同一内核，再用段永平自己的判据做裁决——而「拿段永平的标准去评活体库外人物」这一步，库内任何单一段落都没写。
- 满足硬要求②：对撞纳瓦尔「专长不可培训」+ Vida「方法公开后 alpha 衰减」+ Serenity「信息合成赚认知差」三者与段永平「投资 vs 投机/能力圈/安全边际是了解度」，跨经典与当代、跨价投与投机综合。
- 满足硬要求③（同构且自足）：把库内「公开即失效」的元洞见迁移到判断一个库外活体人物的方法属性——库里没有现成答案；该迁移只需真懂「alpha 来源」原则即可完成，不依赖库外冷知识。
- 死记答案长什么样：「段永平讲价值投资要安全边际、买好公司长期持有；Serenity 讲卡脖子供应链 OSINT、官方确认前买入。两人方法不同，一个稳健一个激进。」——两套并列复述，回避「是否同根」，不敢替段永平下投资/投机判定。
- 真懂答案关键点（须超出任一单一 source 段落）：指出两者底层同源（alpha 都来自不可被公开复制的认知差，只是载体不同、持有期不同）；再用段永平自己的标准裁决（胜率认知是否真在能力圈内达 90%+，否则其高换手/期权/借款抵押避税、依赖未审计自述、利益冲突会被归入投机端）；关键看「是否骨子里相信而非照着念、恐惧程度是否与了解程度成反比」。
- 追问示例：①「段永平说『靠运气赚到的钱多半早晚凭实力亏掉』，而该战绩本人承认 HFT 抢跑其发帖、存在利益冲突——你怎么用库内的『幸存者偏差』和『反馈周期以十年计』给它定性？需要什么证据才能区分实力还是运气？」②「如果这套 OSINT 方法明天被所有人学会写进书里，按纳瓦尔和 Vida 的逻辑会发生什么？这是否反过来证明了『大道理能公开是因为它不失效、术值钱是因为不能公开』？」
