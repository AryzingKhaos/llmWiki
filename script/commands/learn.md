# learn

## 基础要求

目的：

把一份需要深入理解的资料（一本书、一篇论文、一篇文章、一部视频、一段音频），按照斯科特·扬《如何高效学习》中的**整体性学习（Holistic Learning）方法论**，编译成一套"高效学习文档"。产出的不是普通摘要，而是一份帮助学习者真正掌握知识的学习工程文件：包含知识的结构/模型/高速公路、五阶段学习材料（获取、理解、拓展、纠错、应用）、测试设计与薄弱环节定位、记忆技术应用（联想法、比喻法、内在化）以及费曼式解释。

命令格式：

- `learn [输入路径]`

示例：

- `learn improveWiki/raw/深度工作.pdf`
- `learn investWiki/raw/2026/某论文.pdf`
- `learn aiagentWiki/raw/某视频.mp4`

适用场景：

- 想系统性地"学会"一份材料，而不只是存档或摘要（存档/摘要请用 `ingest`）
- 需要为一本书/一门课建立可复习、可自测、可实践的学习文档
- 材料是视频或音频，需要先转成文字再学习

输入：

- 一个文件路径（**唯一必填项**）。可以是文本类（md/txt/pdf/epub/html 等），也可以是视频/音频（mp4/mov/mp3/m4a/wav 等）。

输入要求：

- **`libraryName` 从路径自行推断**：取路径中第一个形如 `xxxWiki/` 的目录段作为 `libraryName`。若无法从路径推断出任何 library，必须提示用户指定这份学习文档应归入哪个 library，不能擅自决定。
- 若输入文件不在对应 library 的 `raw/` 目录下，必须先明确警告用户，再继续流程。
- **视频/音频输入**：必须先用 `video2text` skill 将其转写为文字稿（B 站链接可用 `bili-video-to-text` skill），以转写出的文字稿作为本命令的实际输入语料；转写稿路径必须记录在学习文档的元信息中。
- 开始编译前，必须按"询问规则"与用户确认学习目标；用户未回复、拒绝回答或无法提供时，按默认值执行。

输出结果：

- 输出目录固定为：`[libraryName]/learning/`
- 短材料（单篇文章、论文）：单个文件 `[libraryName]/learning/[yyyyMM]-[中文名]-高效学习.md`
- 长材料（整本书、多章节课程）：新建文件夹 `[libraryName]/learning/[yyyyMM]-[中文名]/`，内含：
  - `00-总览.md`（结构/模型/高速公路 + 拓展索引 + 测试总设计 + 各章文档目录）
  - `01-[章节名].md`、`02-[章节名].md` …（每章一份学习文档）
- 更新 `[libraryName]/learning/index.md`（学习文档索引，没有则新建；格式参照 `wiki/index.md` 的条目格式：链接行 + `Source Path` + 100 字以内简短摘要）

## 方法论速览（本命令的理论依据）

以下概念来自《如何高效学习》，是编写学习文档时必须贯彻的框架：

1. **整体性学习三要素**：
   - **结构**：一系列紧密联系的知识网络。学科内部知识点之间的连接越多、越熟悉，结构越强。
   - **模型**：结构的压缩快照，是结构的种子（如"目录之于整本书"）。用模型可以快速把握整体。
   - **高速公路**：结构与结构**之间**的联系，即用一个领域/生活经验去模拟另一个领域，是创造力和长期记忆的来源。
2. **学习的顺序**：获取 → 理解 → 拓展 → 纠错 → 应用（→ 测试贯穿每个阶段）。任何一个阶段做得差都会成为薄弱环节。
3. **五种信息类型**及其对应技术：
   - **随意信息**（无逻辑的事实/日期/定义）→ 联想法、挂钩法、信息压缩法
   - **观点信息**（存在争论的信息）→ 快速获取大量信息找模式、图表法
   - **过程信息**（怎么做的信息）→ 内在化、比喻法、图表法、模型纠错，重在反复练习
   - **具体信息**（可感知的信息）→ 内在化（多感官联系）
   - **抽象信息**（缺少感官联系但逻辑性强）→ 比喻法、内在化（移到具体层次）、模型纠错
   - 尽量把弱结构（随意信息）转化为强结构（具体/过程信息）。
4. **核心技术**：比喻法（在不熟悉的知识和熟悉的经验之间架桥）、内在化（给知识加上图像、声音、触感、情感）、联想法/挂钩法/压缩法（对付随意信息）、图表法、模型纠错、以项目为基础的学习。
5. **费曼技巧**：选择概念 → 假设教给一名新生并写下解释 → 卡壳处回到原材料 → 用更简单的语言和比喻重述。不翻书就能讲清楚，才算真正理解。
6. **纠错**：比喻和类比天然"跛脚"，必须显式指出比喻与本体的**不同点**和失效边界，删除无效联系。
7. **找出薄弱环节**：成绩不好不等于"笨"，而是获取/理解/拓展/纠错/应用中某个具体环节薄弱。测试的目的就是定位薄弱的知识点与薄弱的环节。

## 执行要求

1. **预处理与体量判断**：
   - 视频/音频先经 `video2text`（或 `bili-video-to-text`）转成文字稿。
   - 读取输入语料，统计正文字数。≤ 5000 中文字（或 ≤ 2500 英文单词）为短材料；超过为长材料，必须走"长材料处理"流程（subagent 分章节处理）。
2. **按"询问规则"确认学习目标**，然后按"学习文档模板"编写文档。
3. **信息类型判定先行**：对每个模块（章节/主题），先判定其主要信息类型（随意/观点/过程/具体/抽象，可以是组合），并据此决定该模块的展示方式：
   - 随意信息 → 列出事实清单 + 为每条设计联想/挂钩/压缩记忆方案
   - 观点信息 → 图表法呈现各方观点对比（表格或 mermaid），提炼争论焦点
   - 过程信息 → 步骤流程图（mermaid）+ 内在化描述 + 练习任务
   - 具体信息 → 场景化描述，调动多感官
   - 抽象信息 → 必须配比喻，把抽象概念拉到具体层次，再给出严格定义
4. **重视比喻和类比**：每个核心概念至少配一个比喻或类比，优先取自日常生活经验。所有比喻都必须在"纠错"部分登记不同点。
5. **拓展做成索引**：拓展部分不展开长篇内容，而是做成三类索引（深度拓展/横向拓展/纵向拓展），每条给出"拓展方向 + 一句话说明 + 可去哪里查（本仓库已有 wiki 条目、原书章节、外部方向）"。若本 library 或其他 library 的 `wiki/index.md` 中存在相关条目，优先链接仓库内条目。
6. **测试必须具体可执行**：不能只写"多做题"。必须依据知识本身设计出题目文本（问答题、辨析题、复述任务、实操任务），并给出"答错说明哪个知识点薄弱、对应哪个学习环节、该用什么技术补救"的对照表。
7. **忠实性边界**：
   - 关于原材料的**事实、观点、结论、数据、人名、引文**，必须在输入语料中真实存在，禁止捏造。
   - **比喻、类比、联想、挂钩、测试题、实践规划、费曼解释**属于本命令要求生成的教学性内容，允许创作，但必须与原材料的含义一致，且在文档中能明确区分"原文说了什么"与"为帮助学习而构造了什么"。
8. **主语言必须是简体中文**。
9. 文档头部（第一个 `## ` 之前）必须包含元信息 bullet 区：`Source Path: <相对仓库根目录的路径>`、信息类型判定结果、学习目标、（若有）转写稿路径、`Confidence` 说明。若源文件不在 `[libraryName]/raw/` 下，额外加一行 `Warning: source file is outside [libraryName]/raw/`。
10. 文件命名规则：`[yyyyMM]-[中文名]`，重名时在末尾加"-2""-3"。
11. 完成后更新 `[libraryName]/learning/index.md`。

## 学习文档模板

每份学习文档（短材料的单文件，或长材料的每章文件）必须包含以下六节；长材料的"一、知识地图"与"六、测试与薄弱环节定位"需要在 `00-总览.md` 中再做全书级汇总。

### 一、知识地图（结构·模型·高速公路）

- **结构**：列出本材料的知识网络——核心概念清单及概念之间的连接关系（可用 mermaid 图）。
- **模型**：用不超过 300 字或一张图，给出整份知识的压缩快照（"如果只能记住一个模型，记住这个"）。模型本身应当是一个比喻或类比。
- **高速公路**：至少 2 条，把本材料与**其他领域**（学习者的生活经验、本仓库其他 wiki 的知识）连接起来，写明"A 领域的 X 就像 B 领域的 Y，因为……"。
- **信息类型判定表**：每个模块 → 主要信息类型 → 选定的展示方式与技术。

### 二、五阶段学习材料

- **获取**：原始知识的忠实提炼——本材料实际说了什么（事实、主张、步骤），按模块列出。做到简化（剔除无用信息）、容量（抓足关键信息）、速度三者平衡。
- **理解**：**专有名词逐个解释**。提取材料中所有专有名词/术语/人名/理论名，逐条给出：原文语境含义 + 大白话解释 + （必要时）一个例子。做成术语表。
- **拓展**（索引形式）：
  - **深度拓展**：知识的背景探究——结论从何而来？为什么成立？谁提出的？实验/证据是什么？
  - **横向拓展**：同类知识——有哪些相似的结论/方法？同一时期/同一领域还有什么？
  - **纵向拓展**：跨领域联系——本知识与完全不同领域的哪些模式相通？（比喻法和内在化在此使用）
- **纠错**：逐条检视"一、知识地图"和"二、拓展"中用到的每个比喻/类比/高速公路，写明：**比喻在哪些方面成立、在哪些方面失效（不同点）**；同时指出原材料本身可能的适用边界或争议点（仅当原文或常识明显支持时，否则标注 `Confidence: Low`）。
- **应用**：实践规划。把知识拆成可实践/可练习的具体行动，给出一个 14 天～3 个月的实践计划骨架：每项行动 + 触发场景 + 最小可验证点（checkpoint）。学习的终点是"知识因用而获得意义"。

### 三、记忆技术应用

- 对材料中的**随意信息**（清单、数字、步骤名），逐条给出联想法（编故事串联）、挂钩法（数字→形象挂钩）或压缩法（首字母缩写/图像压缩）的具体记忆方案。
- 对最重要的 2～5 个概念做**内在化**：写出一段调动视觉/听觉/触觉/情感的想象脚本（"闭上眼睛，想象……"）。
- 没有明显随意信息的材料，此节可以只做内在化，但必须说明理由。

### 四、费曼式解释

- 选取全文 1～3 个最核心（或最难）的概念，各写一段费曼式讲解：
  - 假设向一名完全没有背景的新生讲解
  - 只用简单直白的语言，禁止使用未在"理解"术语表中解释过的术语
  - 必须含至少一个比喻
  - 结尾用一两句话点出"最容易在哪里卡壳、卡壳了回原文哪里看"

### 五、学习顺序建议

- 给出针对本材料的推荐学习路径（先读哪个模块、什么时候做测试、什么时候开始实践），并注明预计时间投入。

### 六、测试与薄弱环节定位

- **怎么测试**：根据信息类型定义测试方式——随意信息考回忆、观点信息考立场复述与对比、过程信息考实际操作、抽象信息考费曼复述与比喻构造。
- **具体测试内容**：出题。每个模块至少 2 题，全文合计 8～20 题，附参考答案（答案要点即可）。题目必须覆盖五个环节：
  - 获取类：这条知识说了什么？（回忆）
  - 理解类：用自己的话解释术语 X。（复述）
  - 拓展类：知识 X 和 Y（另一模块或另一领域）有什么联系？（联系）
  - 纠错类：比喻 Z 在什么情况下失效？以下说法哪里错了？（辨析）
  - 应用类：给出一个真实场景，要求用本知识解决。（实操）
- **薄弱环节定位表**：一张对照表——"哪类题答错 → 说明哪个知识点薄弱 → 属于哪个环节（获取/理解/拓展/纠错/应用）→ 用什么技术补救（如：理解薄弱→放慢速度回到原文；拓展薄弱→练比喻法；纠错薄弱→做模型纠错；应用薄弱→走出书本去实践）"。

## 长材料处理（subagent 分章节）

1. 长材料（> 5000 中文字或 > 2500 英文单词）必须分章节处理，禁止单次整体压缩。
2. 主 AI 先完整浏览材料，列出章节/模块清单并编号（优先按原文目录切分；无目录则按主题切分，每块不超过 5000 中文字，超过则继续拆分子模块）。
3. 主 AI 先向用户复述章节清单和处理计划（每章交给哪个 subagent、产出哪个文件），然后再开始执行。
4. **每章一个 subagent**，各 subagent 只处理自己的章节，按"学习文档模板"产出该章的 `NN-[章节名].md`；subagent 之间不得跨章节合并。
5. subagent 完成后，主 AI 必须逐一回收检查：是否遗漏模块、是否捏造原文没有的事实、比喻是否都登记了不同点、测试题是否覆盖五个环节。复核通过才算完成。
6. 主 AI 亲自编写 `00-总览.md`：全书级的结构/模型/高速公路、跨章节的高速公路与横向拓展索引、全书级测试设计（含跨章综合题 3～5 题）、薄弱环节定位总表、各章文档目录。
7. 章节数 ≤ 2 且总量仅略超阈值时，可以合并为单文件输出，但仍须按章节分派 subagent 处理。
8. 如果"必须使用 subagent"的要求与当前运行环境的上层约束冲突，必须显式说明冲突点并询问用户；冲突未澄清前不得擅自改为单 agent 执行。

## 询问规则

1. 编译开始前，最多问 2 个问题（选择题形式）：
   - **学习目的**：A. 应试/考核（测试部分加重、记忆技术加重） B. 实践应用（应用规划加重、以项目为基础） C. 兴趣理解（比喻/费曼加重、实践计划从简）。默认 B。
   - **重点模块**（仅长材料需要问）：列出章节清单让用户指定重点章节；重点章节的测试题和费曼解释加倍，非重点章节按模板最低要求执行。默认全书均衡。
2. 用户未回复、拒绝回答或无法提供时，一律按默认值执行，不得反复追问。

## 硬性约束

- 只允许写入 `[libraryName]/learning/` 下的文件；不得修改 `raw/` 与 `wiki/sources/` 下的已有文件。
- 原材料相关的事实、观点、数据、引文禁止捏造；教学性生成内容（比喻/联想/测试题/实践规划/费曼解释）必须与原材料含义一致，且与原文内容可区分。
- 每个比喻/类比都必须在"纠错"部分登记不同点，不允许只给比喻不给失效边界。
- 测试题必须是写出来的具体题目，不允许用"可以自测一下"之类的空话代替。
- 拓展部分必须是索引，不允许展开成长篇正文。
- 视频/音频必须先转文字，不得跳过转写直接"凭印象"编写。
- 长材料必须分章节交给 subagent，主 AI 必须复核回收结果。
- 主语言必须是简体中文。
- 不确定的内容必须标注 `Confidence: Low`。
- 如果本规范与当前运行环境的上层约束冲突，必须显式说明冲突点并询问用户如何处理。
