# frontier

## 基础要求

目的：

盘点一个 library 所覆盖领域的「争论地图」与「领域前沿」，回答三个问题：这个领域里什么争论最凶？当前的最前沿在哪里？库内的覆盖落后在哪？

本命令分两部分：第一部分从库内提炼争论并按激烈程度排序（纯库内）；第二部分联网对照当前前沿（必须联网——库内资料是书籍与历史文章，"最前沿"无法纯靠库内回答）。分析问题的方法论以库内为基，前沿事实来自库外，两者必须明确区分。

命令格式：

- `frontier [libraryName] [可选:关注的方面]`

示例：

- `frontier investWiki`
- `frontier aiagentWiki 记忆与检索`

输入：

- 一个 `libraryName`
- 一个可选的「关注的方面」

输入要求：

- 必须显式提供 `libraryName`，不能省略。如果省略，必须提示用户补充，否则不能进行之后的步骤。
- 「关注的方面」可选：提供则收敛范围；不提供则覆盖全库。
- 若指定方面在库内无任何沉淀，如实告知并询问用户是改为全库还是先补资料，不得硬做。

输出结果：

- 一份「前沿与争论」报告，存 `[libraryName]/wiki/fragment/[yyyyMM]-前沿与争论-[方面slug|全库].md`
- 终端输出报告摘要（争论排行榜 + 前沿要点 + 覆盖缺口）

## 执行要求

### 第一部分·库内争论地图（纯库内）

1. 争论入口按以下优先级收集：
	- `wiki/concepts/*.md` 各页的 `## 争议与冲突` 节（首选入口，concept 命令已做过跨源验证）
	- `wiki/concepts/index.md` 的候选概念中的张力型条目
	- `wiki/sources/*.md` 双向链接说明文字中的对立表述
	- 如有必要，回读相关 sources 页正文确认争论的双方表述
2. 汇总争论清单，每条必须包含：争论命题（一句话，写成可裁决的问题）、两派代表与库内出处链接、各派证据强度、库内是否已有某源给出裁决。
3. 按「激烈程度」排序，三个维度综合：
	- **双方证据强度**：两边都是强源（一手原典/系统方法论）才算真凶；一强一弱的是"教学案例"不是战场。
	- **现实赌注**：按这个问题判断错了，实际代价有多大。
	- **未决程度**：库内是否已有裁决、裁决是否被后续来源挑战。
4. 争论必须真实出自库内、逐条附出处；不得制造伪争论，不得把"侧重点不同"包装成争论。

### 第二部分·联网前沿对照

5. 对排行榜上的争论（至少 top 5，全部更好）逐条联网调研当前状态，归入三类之一：
	- **已被裁决**：时代/数据/实践已给出答案（写明裁决方向与依据）
	- **仍在进行**：列出当前两派的最新代表与论据
	- **有新证据**：库内成书后出现的新事实改变了力量对比
6. 反向调研：该领域**当前**最热的争论与前沿方向中，哪些库内完全没有覆盖？逐条列为「覆盖缺口」，并给出建议补充的 raw 类型（书/论文/备忘录/一手访谈等）。
7. 所有联网信息必须附可核验的来源 URL；库内立场与库外现状必须明确区分表述（"库内：X（出处）；库外现状：Y（来源）"），不得混写。
8. 时效标注：报告头部与所有前沿判断必须带「截至 yyyy-MM」。

### 报告与留存

9. 报告结构固定为：元信息（library、方面、截至年月、Confidence）→ `## 争论排行榜`（按激烈度降序，每条含两派/库内立场/库外现状/一句话点评）→ `## 领域前沿速览` → `## 覆盖缺口`（建议补充的 raw 清单）。
10. 同一「方面」再次执行时，新建当月新文件，不修改旧报告——历史报告保留，可对照"争论如何随时间演化"。
11. 报告完成后更新 `wiki/index.md` 的 fragment 相关条目（如该库 index 有 fragment 区块的惯例则遵循之）。

## 硬性约束

- **不回写**：本命令唯一落盘产物是 fragment 报告；严禁修改 `wiki/concepts/`、`wiki/sources/`、`concepts/index.md` 的任何内容。前沿信息时效短，不得沉淀进长期页面；若某条前沿信息值得入库，正确路径是把相应材料放入 raw 后走 ingest。
- 第一部分的争论必须真实出自库内，逐条附出处。
- 第二部分的库外信息必须附来源 URL，且与库内立场明确区分。
- 分析方法论必须以库内为基；不得用库外方法论否定库内框架而不加说明。
- 覆盖缺口只做建议，不自动下载、不自动 ingest（proposal-only）。
- 若某争论的联网调研结果不充分或来源可信度低，必须标注 `Confidence: Low`。
- 禁止执行复制、剪切、移动或重命名 raw 文件的操作。
- 报告主语言必须是简体中文。

## 与其他命令的关系

- 上游：`concept`（争议与冲突节是第一部分的主要供料）；`lint` 的矛盾检测结果可作补充入口。
- 下游：报告的「覆盖缺口」是补料清单（可手动登记进 uningest 流程）；「争论排行榜」是 `probe` 备考期构建跨源张力题的现成题源。
